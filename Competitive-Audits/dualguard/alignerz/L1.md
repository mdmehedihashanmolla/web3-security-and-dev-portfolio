# [L-01] Ineffective `pause` mechanism fails to block NFT transfers during emergencies

Severity: Low
Category: Incident Response / Logic

Platform: DualGuard
Protocol: AlignerZ
Ecosystem: EVM
Language: Solidity

---

## Summary

The `pause()` function is intended to act as an emergency circuit breaker for the protocol. While it successfully blocks new mints and burns, it fails to halt standard token transfers. If the protocol is exploited or experiences a critical bug, attackers can still freely move or sell their vesting NFTs, rendering the emergency stop completely ineffective for containing damage.

---

## Vulnerability Details

The `AlignerzNFT` contract inherits from `ERC721A` and OpenZeppelin's `Pausable`. The contract exposes `pause()` and `unpause()` functions, allowing the owner to freeze operations during an emergency.

The developer correctly applied the `whenNotPaused` modifier to the custom `mint()` and `burn()` functions:

```solidity
function mint(address to) external whenNotPaused onlyMinter returns (uint256) { ... }
function burn(uint256 tokenId) external whenNotPaused onlyMinter { ... }
```

However, the standard ERC721 transfer functions (`transferFrom`, `safeTransferFrom`) are inherited directly from `ERC721A`. By default, these underlying functions do not check the `Pausable` state.

Because `AlignerzNFT` never overrides these transfer functions or the internal transfer hooks to enforce the pause, the contract's `paused` flag is entirely ignored whenever a user moves an NFT.

---

## Root Cause

The contract fails to override the `_beforeTokenTransfers` hook to enforce the `whenNotPaused` constraint. As a result, the inherited `ERC721A` transfer functions execute without ever verifying if the contract is paused.

---

## Impact

The emergency freeze functionality provides a false sense of security. In a live incident response scenario (e.g., a permissionless exploit in the broader protocol), the admin will trigger the pause expecting to lock all state. Instead, attackers will remain fully capable of laundering compromised vesting positions, transferring NFTs to fresh wallets, or dumping them on secondary markets.

---

## Recommendation

The standard and most gas-efficient way to pause all ERC721 transfers is to override the internal `_beforeTokenTransfers` hook provided by `ERC721A`. By applying the `whenNotPaused` modifier to this single hook, all token movements (including mints, burns, and transfers) will be strictly gated by the pause state.

Add the following override to `AlignerzNFT.sol`:

```solidity
/**
 * @notice Hook that is called before any token transfer. This includes minting and burning.
 * @dev Enforces the pause state on all token movements.
 */
function _beforeTokenTransfers(
    address from,
    address to,
    uint256 startTokenId,
    uint256 quantity
) internal virtual override whenNotPaused {
    super._beforeTokenTransfers(from, to, startTokenId, quantity);
}
```
